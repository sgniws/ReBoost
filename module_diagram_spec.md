# 抽象模块图绘制说明（供 AI 绘图使用）

> 本文档仅描述概念层面的模块与关系，不涉及具体项目路径、配置或数据集名称。用于指导生成系统架构/数据流图。

---

## 一、图示类型与层次

- **图示类型**：系统架构图，包含**数据流**（前向与损失反传）与**控制流**（训练循环与策略触发）。
- **建议分层**（自上而下或自左而右均可）：
  1. **输入层**：多模态原始输入与标签
  2. **特征层**：每模态独立特征提取
  3. **预测层**：每模态的预测头（可多个）与可选共享头
  4. **融合层**：多模态预测的聚合
  5. **损失与监控层**：主损失、辅助损失、置信度
  6. **策略与调度层**：自适应头分配与训练循环

---

## 二、节点定义（抽象名称与职责）

| 节点 ID | 显示名称（建议） | 职责简述 |
|--------|------------------|----------|
| **Input** | 多模态输入 | 提供多路模态原始数据与标签 |
| **Backbone** | 模态骨干（×M） | 每个模态独立的编码-解码特征提取，输出该模态的特征图 |
| **Head** | 模态预测头（可多个/模态） | 将单模态特征映射为预测 logits；同一模态可有多个头 |
| **SharedHead** | 共享预测头 | 所有模态、所有头共用的最后一层映射（可选模块） |
| **Fusion** | 融合器 | 将各模态的预测（或各模态内多头求和后的预测）聚合为最终预测 |
| **MainLoss** | 主损失 | 在融合预测与标签上计算的任务损失 |
| **AuxLoss** | 辅助损失 | 各模态上的残差损失、联合预测损失、前序预测损失等 |
| **Confidence** | 置信度计算 | 根据各模态的预测与标签，计算每个模态的置信度标量 |
| **ACA** | 自适应头分配 | 按置信度差距判断是否为最弱模态添加新头 |
| **TrainLoop** | 训练循环 | 驱动前向、损失、反向、参数更新，并周期触发置信度与 ACA |

---

## 三、边定义（流向与含义）

### 3.1 前向数据流

| 起点 | 终点 | 含义 |
|------|------|------|
| Input | Backbone（每个） | 各模态数据送入对应骨干 |
| Backbone（每个） | Head（该模态下所有头） | 骨干输出特征送入该模态的每个预测头 |
| Head（每个） | SharedHead（若存在） | 头的中间输出经共享头得到 logits |
| Head 或 SharedHead 输出 | Fusion | 每模态的 logits（或多头求和后的 logits）送入融合器 |
| Fusion | （输出） | 得到最终预测 |

### 3.2 损失与梯度流

| 起点 | 终点 | 含义 |
|------|------|------|
| Fusion 输出 + Input（标签） | MainLoss | 主损失依赖融合预测与标签 |
| 各模态 Head 输出 + Input（标签） | AuxLoss | 辅助损失依赖各模态的单独/联合/前序预测与标签 |
| MainLoss + AuxLoss | TrainLoop | 总损失用于反向传播与参数更新 |

### 3.3 监控与策略流

| 起点 | 终点 | 含义 |
|------|------|------|
| 各模态预测 + Input（标签） | Confidence | 用各模态预测与标签计算置信度 |
| Confidence | ACA | 置信度分数输入自适应头分配策略 |
| ACA | Head（某模态） | 策略决定为某模态增加新头（动态扩展） |
| TrainLoop | ACA | 训练循环按周期触发置信度计算与 ACA |

---

## 四、绘图布局建议

1. **多模态对称**：M 个模态的 Backbone 与 Head 可横向并排，结构对称，便于看出“每模态独立、融合居中”。
2. **双路损失**：MainLoss 与 AuxLoss 可并列画在 Fusion 与各 Head 下方，汇聚到 TrainLoop 或“总损失”。
3. **策略侧路**：Confidence 与 ACA 可画在主体前向路径的侧面（例如右侧），用虚线或不同颜色表示“周期触发、不每步执行”。
4. **动态头**：Head 可画成“每模态一个框，框内可含多个子头”，ACA 的箭头指向该框表示“在此模态下增加一个子头”。
5. **共享头**：SharedHead 若存在，可画在 Head 与 Fusion 之间，用一条竖线或一个共享框连接所有模态的 Head 输出，再汇入 Fusion。

---

## 五、简化版（仅核心前向）

若只画“前向数据流”简化图，可只保留：

- **Input** → **Backbone（×M）** → **Head（×M，每模态可多子头）** → **SharedHead（可选）** → **Fusion** → **最终预测**

并标注：**置信度**由各模态预测与标签得到；**ACA** 根据置信度为最弱模态**增加新头**；**损失**由融合预测（主损失）与各模态预测（辅助损失）共同组成。

---

## 六、图例建议

- 实线箭头：前向数据流
- 虚线箭头：损失/梯度或周期触发的控制流
- 双线或粗线：融合或总损失汇聚
- 可选图例文字：数据流 / 损失流 / 策略触发
